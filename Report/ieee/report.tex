\begin{abstract}
Accurate line-of-sight (LOS) and non-line-of-sight (NLOS) identification is essential for precise ultra-wideband (UWB) indoor positioning. We study two-path LOS/NLOS classification and path-wise distance estimation on the Decawave DWM1000 dataset. After preprocessing, a valid second path is detected in 40,864 of 41,996 usable samples (97.3\%), producing an expanded two-path dataset of 83,992 rows. On the feature-engineered pipeline, XGBoost gives the best single-model AUC (0.9834), while stacked ensembling attains the highest accuracy (0.9415) with marginal AUC change. A separate raw-CIR CNN+Transformer pipeline reaches AUC 0.9821 on the original split, improving to 0.9845 with CIR augmentation, but this deep-learning result is not directly comparable to the ML results because it uses a different pipeline and split. For regression, path~1 is moderately predictable (best RMSE 1.3706\,m, $R^2=0.6698$), whereas path~2 attains near-perfect scores because its target is derived from input features supplied to the model. These results support tree-based supervised models for this dataset, while unsupervised clustering remains weak.
\end{abstract}

\begin{IEEEkeywords}
UWB, LOS/NLOS classification, channel impulse response, CNN, Transformer, indoor positioning, distance estimation

\section{Introduction}

Ultra-wideband (UWB) technology has emerged as a leading solution for centimeter-level indoor positioning due to its high time resolution and resistance to multipath fading~\cite{b1}. However, NLOS propagation conditions cause systematic ranging errors that severely degrade positioning accuracy. Identifying whether a signal travels via LOS or NLOS is therefore critical for mitigating these errors~\cite{b2}.

The channel impulse response (CIR) recorded by UWB receivers encodes rich information about the propagation environment. Traditional approaches extract hand-crafted features such as rise time, kurtosis, and energy ratios from the CIR, then apply classifiers like support vector machines (SVM), k-nearest neighbors, or random forests~\cite{b3,b4}. While effective, these methods rely on domain expertise for feature selection and may miss subtle signal patterns.

Recent advances in deep learning offer an alternative: learning discriminative features directly from raw waveforms. Convolutional neural networks (CNNs) have been applied to UWB CIR classification~\cite{b5}, but they capture only local patterns. Transformer architectures~\cite{b6} excel at modeling long-range dependencies through self-attention, and recent work has demonstrated their effectiveness for UWB NLOS detection specifically~\cite{b10,b11}. This makes hybrid CNN-Transformer designs well-suited for analyzing the full CIR waveform where multipath components may be widely separated.

In this paper, we make the following contributions:
\begin{itemize}
    \item A two-path extraction and labeling framework that extends binary LOS/NLOS classification to the two dominant signal paths in each CIR measurement.
    \item A feature engineering pipeline producing 25 per-path features for classical ML models, with a compact RFECV-selected subset also examined.
    \item A hybrid 1D-CNN + Transformer architecture that operates directly on the raw 1016-sample CIR waveform in a separate pre-computed pipeline.
    \item Distance estimation models for both dominant paths, interpreted with attention to the analytic constraint in the path~2 target definition.
\end{itemize}

\section{Dataset and Data Preparation}

\subsection{Dataset Description}

We use the publicly available UWB LOS/NLOS dataset~\cite{b7} collected using Decawave DWM1000 modules across seven distinct indoor environments. The dataset contains 42,000 samples (21,000 LOS and 21,000 NLOS), each comprising 14 scalar features and a 1016-sample CIR recorded at 1\,ns resolution. Table~\ref{tab:features} summarizes the key features.

\subsection{Data Preparation}

Our preprocessing pipeline performs the following steps:

\textbf{CIR normalization.} Each CIR sample is divided by its corresponding RXPACC value to normalize the preamble accumulation effect.

\textbf{Constant/metadata removal.} Three raw features (CH, BITRATE, PRFR) are constant and removed. We additionally drop loader metadata (ENV\_ID) to avoid using environment identifiers as predictors, leaving the 11 scalar features used downstream.

\textbf{Feature scaling.} The 11 remaining scalar features are standardized using zero-mean unit-variance scaling (StandardScaler).

\textbf{NaN/Inf handling.} Any NaN or Inf values introduced during CIR normalization are replaced with zero.

\textbf{Train/test split.} After removing 4 degenerate samples, the usable set has 41,996 rows and is split stratified 80/20 into 33,596 training and 8,400 test samples.

Fig.~\ref{fig:distributions} shows the distribution of key features by LOS/NLOS class, revealing clear separability for several features. Correlation analysis shows that FP\_AMP1--3 are highly correlated ($r>0.9$), as are STDEV\_NOISE and MAX\_NOISE ($r\approx0.95$).

\section{Proposed Methodology}

\subsection{Two-Path Extraction}

UWB CIR measurements often contain multiple propagation paths. The two shortest (dominant) paths carry the most information for positioning: if the first path is LOS, the second is necessarily NLOS (reflected); if the first path is NLOS, both paths are NLOS. Algorithm~\ref{alg:peak} describes our peak detection procedure.

\begin{algorithm}[t]
\caption{Two-Path Peak Detection}
\label{alg:peak}
\begin{algorithmic}[1]
\REQUIRE CIR waveform $c[0..1015]$, first path index $\mathrm{FP\_IDX}$
\ENSURE Path indices $p_1, p_2$ and amplitudes $a_1, a_2$
\STATE $p_1 \leftarrow \arg\max_{j \in [\mathrm{FP\_IDX}-10, \mathrm{FP\_IDX}+10]} c[j]$
\STATE $a_1 \leftarrow c[p_1]$
\STATE Smooth $c$ with a 3-sample moving average to obtain $\tilde{c}$
\STATE Find peaks in $\tilde{c}$ with height $> 0.02 \cdot a_1$, distance $> 15$, prominence $> 0.02 \cdot a_1$
\STATE Remove peaks within $\pm 15$ samples of $p_1$
\STATE $p_2 \leftarrow$ remaining peak with maximum prominence (fallback: max amplitude)
\STATE $a_2 \leftarrow c[p_2]$ (from original unsmoothed CIR)
\RETURN $p_1, a_1, p_2, a_2$
\end{algorithmic}
\end{algorithm}

After preprocessing, 41,996 samples remain usable for two-path extraction, and a second path is detected in 40,864 of them (97.3\%). Fig.~\ref{fig:cir} shows example CIR waveforms with detected peaks.

\textbf{Two-path labeling.} For each usable original sample, we generate two rows: path~1 retains the original LOS/NLOS label, while path~2 is always labeled NLOS: if path~1 is LOS, then path~2 is necessarily a reflected component, and if path~1 is itself NLOS, then path~2 is also NLOS. This yields an expanded dataset of 83,992 rows with a 25\%/75\% LOS/NLOS class imbalance.

The classical ML study includes Logistic Regression (LR), Support Vector Machine (SVM), Random Forest (RF)~\cite{b9}, Gradient Boosted Trees (GBT), and XGBoost (XGB)~\cite{b8}, followed by simple averaging and stacked ensembles over the strongest tree-based models.

\subsection{End-to-End Deep Learning Pipeline}

We propose a hybrid 1D-CNN + Transformer architecture that operates directly on the raw 1016-sample CIR waveform. Algorithm~\ref{alg:training} describes the training procedure.

\textbf{Architecture.} The model consists of three components:

\textit{1) CNN encoder:} A multi-scale first convolutional stage (parallel kernels 3/7/15) is followed by MaxPool(2), halving the sequence from 1016 to 508. Two subsequent residual Conv1D blocks~\cite{b14} (kernels 5 and 3) with max-pooling progressively reduce this to 127 feature vectors of dimension 128.

\textit{2) Transformer encoder:} Two Transformer encoder layers with 4-head self-attention model global dependencies across the CNN-encoded sequence. Sinusoidal positional encoding preserves temporal ordering. The self-attention mechanism allows the model to relate distant CIR components, capturing multipath relationships that span the full waveform.

\textit{3) Classification head:} Global average pooling reduces the transformer output to a single 128-dimensional vector, which is concatenated with 11 scaled scalar features and passed through a two-layer MLP (hidden size 64) to produce a binary logit.

\begin{table}[t]
\caption{CNN-Transformer Hyperparameters}
\label{tab:hyperparams}
\centering
\begin{tabular}{@{}ll@{}}
\toprule
\textbf{Parameter} & \textbf{Value} \\
\midrule
CNN channels & 32 $\rightarrow$ 64 $\rightarrow$ 128 \\
CNN kernels & 3/7/15 (parallel), then 5 and 3 \\
Transformer layers & 2 \\
Attention heads & 4 \\
Feedforward dim & 256 \\
MLP hidden & 64 \\
Dropout & 0.1 \\
Optimizer & Adam~\cite{b15} (lr = $1 \times 10^{-3}$) \\
Loss & BCEWithLogitsLoss \\
Label smoothing & $\varepsilon = 0.05$ ($y' = 0.975/0.025$)~\cite{b12} \\
Batch size & 256 \\
Early stopping & patience = 5 \\
Device & Auto-select (CUDA/MPS/CPU) \\
\bottomrule
\end{tabular}
\end{table}

\begin{algorithm}[t]
\caption{CNN-Transformer Training}
\label{alg:training}
\begin{algorithmic}[1]
\REQUIRE Training set $\{(\mathbf{c}_i, \mathbf{s}_i, y_i)\}_{i=1}^{N}$ (CIR, scalars, label)
\ENSURE Trained model $f_\theta$
\STATE Initialize $f_\theta$ with random weights
\STATE $\text{best\_loss} \leftarrow \infty$; $\text{patience\_count} \leftarrow 0$
\FOR{epoch = 1 to 50}
    \FOR{each mini-batch $B$}
        \STATE $y'_B \leftarrow y_B \cdot 0.95 + 0.025$ \COMMENT{label smoothing, $\varepsilon{=}0.05$}
        \STATE $\hat{y} \leftarrow f_\theta(\mathbf{c}_B, \mathbf{s}_B)$
        \STATE $\mathcal{L} \leftarrow \text{BCEWithLogits}(\hat{y}, y'_B)$
        \STATE Update $\theta$ via Adam with $\nabla_\theta \mathcal{L}$
    \ENDFOR
    \STATE Compute validation loss $\mathcal{L}_\text{val}$
    \IF{$\mathcal{L}_\text{val} < \text{best\_loss}$}
        \STATE $\text{best\_loss} \leftarrow \mathcal{L}_\text{val}$; save $\theta$; reset patience
    \ELSE
        \STATE $\text{patience\_count} \leftarrow \text{patience\_count} + 1$
    \ENDIF
    \IF{$\text{patience\_count} \geq 5$}
        \STATE \textbf{break} (early stopping)
    \ENDIF
\ENDFOR
\RETURN $f_\theta$ with best validation weights
\end{algorithmic}
\end{algorithm}

The DL model is trained on the original balanced 42,000-sample dataset (not the two-path expanded set), using an 80/20 stratified split in a separate pre-computed raw-CIR pipeline.

Path~1 regression is materially harder than classification because the target RANGE includes environment-dependent NLOS bias~\cite{b3,b4}, with the best results coming from RF/XGB-style tree ensembles at roughly 1.37\,m RMSE and $R^2\approx0.67$. Path~2 appears much easier, but this must be interpreted carefully: the target is defined as $\texttt{RANGE} + (\texttt{path2\_idx} - \texttt{FP\_IDX})\times 0.2998$\,m/ns (speed of light), while \texttt{RANGE} is also supplied as an input. The near-perfect path~2 scores therefore reflect a strong analytical constraint in the target construction rather than an entirely independent prediction task. \section{Discussion}

\subsection{ML vs. Deep Learning Comparison}

The principal supervised finding is that feature-engineered tree models perform well on the expanded two-path task: XGBoost reaches AUC 0.9834, and stacked ensembling yields only marginal improvement. The CNN+Transformer also performs strongly, reaching AUC 0.9821 on the original split and 0.9845 with CIR augmentation, but this ML-vs-DL comparison is not direct because the deep model is trained and evaluated in a separate raw-CIR pipeline on the original split. Fig.~\ref{fig:comparison} summarizes the model comparison.

Accordingly, we treat the DL results as complementary evidence that raw-CIR learning is viable, rather than as evidence that DL surpasses ML on an identical benchmark. Within the notebook experiments, SMOTE has negligible effect on the ML pipeline, whereas CIR augmentation produces a more pronounced gain for the separate CNN+Transformer pipeline.

\subsection{Error Analysis}

Classification errors primarily occur in ambiguous cases where NLOS signals have strong direct-path-like characteristics or where LOS signals experience significant multipath interference. The gap between LR and tree-based models indicates a nonlinear decision boundary, while the weak SVM result suggests that the current feature representation and tuning are ill-suited for a margin-based classifier.

For distance estimation, path~1 remains the more meaningful regression task because the measured range includes environment-dependent bias that is not trivially recoverable from a single formula. By contrast, path~2 regression benefits from the direct geometric relationship embedded in the target definition and reflects the analytical dependency rather than genuine predictive difficulty.

Unsupervised clustering is weak. KMeans reaches only 0.5906 test accuracy with ARI 0.0269, and DBSCAN fails to discover any clusters, labeling 100\% of samples as noise. These results suggest that LOS/NLOS separation in this feature space is not readily achievable through unsupervised methods alone.

From a practical standpoint, the CNN-Transformer operates on the full 1016-sample CIR plus 11 scalar features, avoiding manual feature engineering, whereas the tabular models require the 25-dimensional engineered feature vector.

\section{Conclusion}

We presented a concise UWB LOS/NLOS classification and distance-estimation study using a feature-engineered two-path pipeline together with a separate raw-CIR CNN+Transformer reference pipeline. The two-path extraction stage finds a valid second path in 97.3\% of usable preprocessed samples and expands the supervised dataset to 83,992 rows.

The key findings are as follows: (1) among directly comparable ML models, XGBoost gives the best single-model AUC (0.9834), while stacked ensembling gives the highest accuracy (0.9415) with marginal AUC change; (2) the CNN+Transformer is competitive, and CIR augmentation improves it further, but this DL result comes from a separate original-split raw-CIR pipeline and is not a strict one-to-one comparison with the ML results; (3) path~1 regression is moderately accurate, whereas path~2 regression is analytically constrained by the way its target is defined; and (4) clustering results are weak and should not be used to claim strong unsupervised separability.

\subsection{Future Work}

Several directions merit further investigation: (1) Ensemble methods combining ML and DL predictions; (2) Cross-environment generalization studies using leave-one-environment-out validation; (3) Model compression and quantization for real-time deployment on embedded UWB receivers; (4) Extension to multi-class scenarios distinguishing between different NLOS propagation mechanisms (diffraction, reflection, scattering).
